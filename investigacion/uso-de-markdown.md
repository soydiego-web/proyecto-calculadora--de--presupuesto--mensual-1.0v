# Documentación: Uso de Markdown en Proyectos de Software

##  ¿Qué es Markdown?

**Markdown** es un lenguaje de marcado ligero que permite escribir
documentos de forma sencilla utilizando una sintaxis fácil de leer y
convertir posteriormente a HTML u otros formatos.\
Se utiliza ampliamente en proyectos de software por su capacidad de
documentar sin complejidad y mantener los archivos versionables.

### ¿Por qué se utiliza en proyectos de software?

-   Permite crear documentación clara y fácil de mantener.
-   Es compatible con plataformas como GitHub, GitLab y Bitbucket.
-   Facilita la escritura sin necesidad de herramientas avanzadas.
-   Se integra perfectamente con sistemas de control de versiones.
-   Puede convertirse a otros formatos como PDF, HTML, DOCX, etc.

##  Ejemplo práctico de uso de Markdown

### 1. Encabezados

``` markdown
# Encabezado 1
## Encabezado 2
### Encabezado 3
```

### 2. Listas

**Listas ordenadas:**

``` markdown
1. Primer punto
2. Segundo punto
3. Tercer punto
```

**Listas sin orden:**

``` markdown
- Elemento A
- Elemento B
- Elemento C
```

### 3. Tablas

``` markdown
| Nombre | Edad | País |
|--------|------|------|
| Ana    | 22   | Perú |
| Carlos | 30   | México |
| Sofía  | 28   | Chile |
```

### 4. Enlaces

``` markdown
[Visitar GitHub](https://github.com)
```

### 5. Imágenes

``` markdown
![Ejemplo de imagen](https://upload.wikimedia.org/wikipedia/commons/4/48/Markdown-mark.svg)
```

##  Ventajas de utilizar Markdown en combinación con GitHub

-   **Control de versiones:** Cada modificación queda registrada
    históricamente.
-   **Visualización inmediata:** GitHub interpreta Markdown
    automáticamente.
-   **Ideal para documentación colaborativa:** Equipos pueden editar y
    mejorar archivos sin conflictos.
-   **Integración con README:** La mayoría de proyectos usan un
    README.md como presentación oficial.
-   **Soporte en issues y pull requests:** GitHub permite usar Markdown
    en comentarios, tareas, checklists y reportes.
-   **Compatibilidad con GitHub Pages:** Markdown se puede convertir en
    sitios web estáticos fácilmente.
-   **Liviano y portable:** No requiere programas externos y funciona en
    cualquier sistema operativo.

------------------------------------------------------------------------

##  Conclusión

Markdown es una herramienta esencial en proyectos de software gracias a
su simplicidad, compatibilidad con GitHub y versatilidad para documentar
proyectos de manera profesional y eficiente.
